# MiniMind Plan mode prompt 模板

本模板用于要求 Codex 在 MiniMind 项目里先计划、再执行。生成最终 prompt 时必须写清具体任务，不保留待填写语句。

````text
你要在 `/home/harry/projects/MiniMind` 处理一次 MiniMind 任务。先进入 Plan mode，只做只读探索，不要编辑文件、不要 apply patch、不要新增报告、不要删除文件、不要修改版本号。

【必须先读】

1. `AGENTS.md`
2. `README.md`
3. `docs/minimind-roadmap.md`
4. 与本轮任务相关的本地文件
5. 必要时只读核验 `/home/harry/references/minimind` 和 `/home/harry/references/learn-minimind`

【只读阶段要确认】

- 当前本地仓库和个人远端状态。
- 本轮任务涉及的真实入口、调用链、数据结构或文档边界。
- 哪些事实来自本地，哪些来自上游，哪些只是学习材料。
- 是否涉及 RTX 5060 Laptop 约 8GB 显存边界。
- 是否需要后续生成实验记录、源码地图或修复报告。

【必须提炼的关键决策】

从真实材料中提炼 2 到 4 个需要用户拍板的决策。优先关注：

- 同步上游源码的范围和可回滚边界。
- 手写复现、直接引用上游、还是先写对照测试。
- 验证深度：静态阅读、单测、单 batch、GPU smoke test。
- 文档沉淀位置：`docs/`、`docs/experiments/`、`docs/reports/`。
- 是否需要版本号或报告；默认文档和 skill 治理不升版。

不要把能通过读文件确认的事实、琐碎文件名、函数名或只有一个合理答案的工程常识交给用户决策。

【每个决策点必须包含】

1. 为什么现在必须决策。
2. 可选方向。
3. 每个方向的影响。
4. 当前基于仓库事实的倾向。
5. 需要用户确认的选择。

【用户选择后】

再输出执行计划，包含文件级改动范围、验证命令、证据路径、风险、回滚边界和未验证项。不要在用户选择前直接实现。
````
